## 1. 计划摘要（可开工定义）

在 **Android 8.0+（API 26+）** 上，交付一款 **完全离线、局域网/WiFi Direct 可达** 的多人相册：**群主嵌入式 HTTP（NanoHTTPD）+ 客户端 OkHttp**；**Room** 存元数据；UI 为 **Compose Material 3**（§3、§4）。

**「可开工」含义**：下列 **阶段 0～1** 的范围、架构边界、界面骨架与测试矩阵已锁定；**A1–A3、B1–B3、C1 已于第 8 节定稿**（代你裁决为推荐项），不阻塞工程搭建与 P2P+HTTP 第一条竖切。母本 `docs_共享相册App需求文档.md` **已修订为 V1.1**，§7.1 / §4.2 / §5.2 等与 autoplan 定稿一致。

## 6. 实施顺序（建议迭代）

1. **连接竖切**：Connection 层 + 群主 IP、端口、热点/AP 配置；真机双机 ping 通。  
2. **HTTP 竖切**：NanoHTTPD `GET /api/photos` + `POST /api/upload` + 静态缩略图路径；OkHttp 客户端打通。  
3. **Room + Repository**：Photo 最小实体；群主权威、成员投影。  
4. **Compose 四屏骨架**：Gallery / Detail / Create / Devices + 全局连接状态（**设备页二维码 B2**：阶段 1 不做，阶段 2 再做）。  
5. **阶段 1 验收用例**：2 台上传+浏览+下载；再扩到 3～4 台压一点并发；验证列表增量 **304/空 body 或 sinceVersion** 路径。  
6. **阶段 2 门禁**：补 WebSocket、事件幂等、标记/评论 API 与 UI、二维码邀请（若仍需要）。
